# Checkbox

Checkboxes can be used to turn an option on or off.

If you have multiple options appearing in a list, you can preserve space by using checkboxes
instead of on/off switches.
If you have a single option, avoid using a checkbox and use an on/off switch instead.

## Demo

export const DemoBlankComponent = ({}) => {
  const [checked, setChecked] = useState(false)
  return <Checkbox checked={checked} onCheckedValueChange={setChecked} />
}

export const DemoCheckedComponent = ({}) => {
  const [checked, setChecked] = useState(true)
  const [partial, setPartial] = useState(false)
  return (
    <Checkbox
      checked={checked}
      partial={partial}
      onCheckedValueChange={setChecked}
      onPartialValueChange={setPartial}
    />
  )
}

export const DemoPartialComponent = ({}) => {
  const [checked, setChecked] = useState(false)
  const [partial, setPartial] = useState(true)
  return (
    <Checkbox
      checked={checked}
      partial={partial}
      onCheckedValueChange={setChecked}
      onPartialValueChange={setPartial}
    />
  )
}

## Basic usage

### Without label on the top

```typescript type=live
<Checkbox
  checked={false}
  partial={false}
  onCheckedValueChange={() => {}}
  onPartialValueChange={() => {}}
  label="label"
/>
```

### With label on the top

```typescript type=live
<CheckboxField
  label="Checkbox field label"
  checked={false}
  partial={false}
  onCheckedValueChange={() => {}}
  onPartialValueChange={() => {}}
/>
```

## Props

Checkbox forwards all unknown props to the base native `<input>` component.
See [`<input type="checkbox">` on MDN](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/input/checkbox)
for more details on which additional props that are available.
